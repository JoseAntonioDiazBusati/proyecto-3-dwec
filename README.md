# Proyecto 3: Mejora y ampliación de un proyecto web
## Fundamentación teórica
En el desarrollo de aplicaciones web, uno de los desafíos más comunes es trabajar con código legado (legacy code), que son sistemas ya existentes que necesitan ser actualizados o mejorados. En muchas ocasiones, los desarrolladores no empiezan un proyecto desde cero, sino que deben ampliar, optimizar o mejorar funcionalidades en aplicaciones que ya han sido creadas, garantizando su mantenimiento y evolución sin comprometer la integridad del código original.

Este proyecto te brinda la oportunidad de enfrentarte a ese reto, aplicando tus conocimientos en manipulación del DOM, eventos, validaciones de formularios y almacenamiento en el navegador. La capacidad para identificar áreas de mejora, diseñar nuevas funcionalidades y optimizar código existente es una habilidad esencial para cualquier desarrollador, y este ejercicio busca replicar una situación típica del día a día en la industria del desarrollo web.

## Objetivos del proyecto
* Ampliar funcionalidades: Identificar áreas de mejora en un proyecto existente y añadir nuevas funcionalidades utilizando eventos, DOM, formularios y almacenamiento en el navegador.
* Optimización de código: Mejorar el rendimiento y la eficiencia del código heredado, aplicando buenas prácticas de desarrollo y refactorización.
* Documentación clara: Mantener una documentación detallada de los cambios realizados, explicando las nuevas funcionalidades implementadas.
* Uso de sintaxis moderna: Demostrar el uso de las características modernas de JavaScript, para escribir un código limpio, eficiente y sostenible.
## Descripción del proyecto
En este proyecto, trabajarás con un código existente de uno de los proyectos que ya has desarrollado en clase. Como desarrolladores web, no solo implementamos funcionalidades que nos solicitan, sino que a menudo tenemos que proponer mejoras y soluciones a nuestros clientes, basándonos en nuestra experiencia y conocimiento de las herramientas disponibles.

Tu objetivo será identificar áreas de mejora en el proyecto elegido y añadir nuevas funcionalidades, aprovechando los conocimientos adquiridos sobre eventos, DOM, formularios y almacenamiento en el navegador.

### Tareas principales:
#### 1. Selección de proyecto base

Elige uno de los proyectos previos desarrollados en clase como base para tu ampliación.

**[Carrito de compra](https://github.com/JoseAntonioDiazBusati/Proyecto-Carrito)**


Al igual que en el mundo real, a veces nos toca pensar en mejoras que podamos sugerir a nuestros clientes. Analiza el proyecto seleccionado e identifica al menos tres áreas de mejora o funcionalidades nuevas que puedas implementar utilizando los conocimientos adquiridos.

Si no tienes claro por dónde empezar, aquí tienes algunos ejemplos de mejoras que podrías implementar, pero si se te ocurren otras ideas, ¡mejor aún!:

* Mejorar la interfaz de usuario para hacerla más intuitiva o atractiva.
* Añadir nuevas validaciones de formularios para aumentar la seguridad o mejorar la experiencia del usuario.
* Implementar funcionalidades de arrastrar y soltar (drag and drop), si es aplicable.
* Optimizar la experiencia del usuario mediante eventos y animaciones.
* Añadir capacidades de búsqueda o filtrado avanzadas.
* Ampliar el uso de almacenamiento en el navegador, como LocalStorage o IndexedDB.
* Implementar nuevas funcionalidades basadas en la lógica del proyecto, que puedan aportar valor a los usuarios o al cliente.
Recuerda, debes proponer al menos tres mejoras que consideres útiles para el proyecto. Tu capacidad para proponer y justificar estas mejoras es parte de la evaluación.

#### 3. Desarrollo e implementación
   Implementa las nuevas funcionalidades en el proyecto seleccionado, asegurándote de seguir las buenas prácticas de desarrollo (código limpio, uso de sintaxis moderna, manejo eficiente de eventos, etc.).

#### 4. Documentación
   Documenta todas las mejoras realizadas, explicando las funcionalidades adicionales y cómo estas mejoran el proyecto original.

Este proyecto no solo pone a prueba tu capacidad para ampliar proyectos existentes, sino que también desarrolla tu habilidad para identificar y proponer mejoras, una habilidad clave en el entorno de desarrollo web.
